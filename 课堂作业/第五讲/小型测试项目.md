# 小型测试项目 - Web登录功能测试

## 项目概述
本项目是一个完整的Web登录功能测试项目，涵盖了从需求分析到测试报告的完整测试流程。

## 测试项目5个阶段

### 1. 需求分析阶段
**工具**: Jira  
**目的**: 录入需求与任务

#### 需求描述
- **功能需求**: 用户能够通过用户名和密码登录系统
- **性能需求**: 登录响应时间不超过3秒
- **安全需求**: 防止SQL注入、暴力破解等攻击
- **兼容性需求**: 支持主流浏览器(Chrome、Firefox、Edge)

#### Jira任务清单
1. **PROJ-001**: 分析登录功能需求
2. **PROJ-002**: 定义测试范围和测试策略
3. **PROJ-003**: 识别测试风险和依赖关系
4. **PROJ-004**: 制定测试计划和时间表

### 4. 缺陷提交阶段
**工具**: Jira / Bugzilla  
**目的**: 报告并跟踪缺陷

#### 缺陷分类标准
- **严重程度**:
  - Blocker: 阻塞性缺陷，系统无法使用
  - Critical: 严重缺陷，核心功能失效
  - Major: 主要缺陷，重要功能异常
  - Minor: 次要缺陷，不影响主要功能

- **优先级**:
  - P1: 立即修复
  - P2: 当前版本修复
  - P3: 下个版本修复
  - P4: 有时间再修复

#### 缺陷报告模板
```
缺陷ID: BUG-001
标题: 用户名为空时仍可登录成功
严重程度: Major
优先级: P2
发现环境: Chrome 120.0, Windows 11
重现步骤:
1. 打开登录页面
2. 密码框输入"password123"
3. 用户名框保持为空
4. 点击登录按钮
预期结果: 显示"请输入用户名"错误提示
实际结果: 登录成功，跳转到主页
```
