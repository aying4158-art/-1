# Web登录功能测试用例

## 测试目标
验证Web登录功能的正确性、安全性和用户体验

## 测试环境
- 浏览器：Chrome, Firefox, Edge
- 操作系统：Windows
- 测试工具：Selenium WebDriver + Python

## 测试用例设计

### TC001 - 正常登录测试
**测试目的**：验证用户使用正确的用户名和密码能够成功登录
**前置条件**：用户已注册，系统正常运行
**测试步骤**：
1. 打开登录页面
2. 输入正确的用户名：admin
3. 输入正确的密码：password123
4. 点击登录按钮
**预期结果**：成功登录，跳转到主页面
**优先级**：高

### TC002 - 用户名为空测试
**测试目的**：验证用户名为空时的错误处理
**前置条件**：系统正常运行
**测试步骤**：
1. 打开登录页面
2. 用户名输入框保持为空
3. 输入密码：password123
4. 点击登录按钮
**预期结果**：显示"请输入用户名"错误提示
**优先级**：高

### TC007 - SQL注入测试
**测试目的**：验证系统对SQL注入攻击的防护
**前置条件**：系统正常运行
**测试步骤**：
1. 打开登录页面
2. 输入用户名：admin' OR '1'='1
3. 输入密码：anything
4. 点击登录按钮
**预期结果**：登录失败，显示错误提示，不能绕过认证
**优先级**：高

### TC008 - 密码显示隐藏测试
**测试目的**：验证密码输入框的安全性
**前置条件**：系统正常运行
**测试步骤**：
1. 打开登录页面
2. 在密码输入框中输入密码
**预期结果**：密码以星号或点号形式显示，不显示明文
**优先级**：中
